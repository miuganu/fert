Fert

Authors
Iuganu Teodora-Madalina & Dulan Dragos Andrei Anul 3, Grupa A6

Abstract

Fert este o aplicatie open source (https://github.com/miuganu/Fert ) ce se doreste a fi un site de feedback pentru studenti. Aceasta aplicatie a fost conceputa in scop pur didactic reprezentand un proiect in cadrul materiei Dezvoltarea aplicatiilor Web la nivel de client.

Introducere

Fert este o aplicatie web pentru colectarea de feedback din partea studentilor in timp real cadrul unui curs, astfel o accesare a aplicatiei propriu-zisenecesita legare utilizatorului. Implicit in cadrul aplicatiei un utilizator(logat) ca si student va avea posibilitatea de a pune intrebari, a raspunde la intreabrile profesorului si pentru a oferi constat feedback daca a inteles sau nu. De asemenea, daca un utilizator este logat ca si profesor poate crea o noua ora unde va putea vedea feedback-ul si de a pune intrebari.

Structura

Pagina de pornire
• Va contine un formular de login unde sunt prezente 2 campuri si anume username si parola, o nepotrivire a acestora va genera diverse mesaje de eroare. De asemeni va exista un buton de Login care permite accesul catre pagina specifica fiecarui utilizator.
Pagina principala va contine
• Header-ul care contine numele aplicatiei(un simplu click va redirectiona utilizatorul catre pagina principala), un sidebar care va facilita interschiparea intreabrile paginile specifice utilizatorilor, spre exemplu pesntru elev vor fi paginile Raspunde, Votare si intreaba. Pe langa acestea se afla si butonul de logout.

In cazul unui student, paginile la care va avea acces sunt:
Pagina Code
• Pagina unde studentul va introduce codul spus de profesor pentru a-si face prezenta si pentru a intra la cursul respectiv.
• Este necesara completarea tuturor campurilor pentru a adauga o licitatie si anume: numele produsului, pretul initial, categoria din care face parte, o imagine in diferite formate, o data de expirare care nu depaseste o luna de zile, si o descriere a produsului.
• Aceasta pagina este prevazuta si cu un buton care permite salvarea licitatiei adaugate.

Pagina Home
• Reprezinta pagina cu toate optiunile dispobile si redirectarile catre restul paginilor.
• De asemenea, este pagina unde studentul poate alege daca a inteles sau nu a inteles topicul prezent.

Pagina Raspunde
• Reprezinta pagina cu intrebarile grila puse de profesor.
• Studentul va putea alege, daca vrea, un raspuns si cu ajutorul butonului submit va trimite alegerea profesorului.

Pagina de intreaba
• Aceasta pagina serveste drept scop neclaritatilor studentului asupra prelegerii. Acesta va putea pune si intrebari anonime.

Pagina de Voteaza
• Pune la dispozitie intrebarile celorlalti studenti pentru a fi votate, astfel avand un numar M de voturi, fiind afisate si profesorului.

In cazul unui profesor, paginile la care va avea acces sunt:
• PEntru a vedea grupele, topicuri si pentru a pune intrebari.

Utilizatorul înregistrat

Se logheaza pe aplicatie folosind un username si o parola. Dupa ce s-a logat are urmatoarele drepturi:
(elev/student)
- adauga o intrebare;
- voteaza intrebari;
- poate oferi feedback in timp real;
(profesor)
-adauga intrebare tip grila;
-poate vedea feedback-ul minimal;

Utilizator fara cont

Utilizatorul fara cont nu va putea accesa aplicatia.

Tehnologii utilizate

Proiectul utilizeaza urmatoarele tehnologii: HTML5 - este un limbaj pentru structurarea si prezentarea continutului pentru World Wide Web, o tehnologie nucleu pentru Internet propusa initial pentru software-ul Opera.

CSS - (Cascading Style Sheets) este un standard pentru formatarea elementelor unui document HTML. Stilurile se pot atasa elementelor HTML prin intermediul unor fisiere externe sau în cadrul documentului, prin elementul "style" si/sau atributul style. CSS se poate utiliza si pentru formatarea elementelor XHTML, XML si SVGL

Javascript - este un limbaj de programare orientat obiect bazat pe conceptul prototipurilor.Este folosit mai ales pentru introducerea unor functionalitati în paginile web, codul Javascript din aceste pagini fiind rulat de catre browser. Limbajul este binecunoscut pentru folosirea sa in construirea siturilor web, dar este folosit si pentru acesul la obiecte incastrate (embedded objects) în alte aplicații. A fost dezvoltat initial de către Brendan Eich de la Netscape Communications Corporation sub numele de Mocha, apoi LiveScript, și denumit în final JavaScript.

Servicii

1. Github
• GitHub este un serviciu de gazduire web pentru proiecte de dezvoltare a software-ului care utilizeaza sistemul de control al versiunilor Git. GitHub ofera planuri tarifare pentru depozite private si conturi gratuite pentru proiecte open source.

2. Firebase - este un serviciu de backend oferit de cei de la Google care contine Realtime Database,Firestore,Auth,Hosting si multe altele.

• Realtime Database - vom folosi acest serviciu pentru a primi cat mai rapid datele din baza de date. Acesta se foloseste de WebSocket-uri ceea ce ajuta la distribuirea datelor intr-o maniera mai eficienta.

• Auth - pentru autentificarea userilor prin email si parola.
